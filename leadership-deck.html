Key Clarifications
Common QuestionAnswer
Does Google provide the API?No. Merchants expose APIs. Google's Gemini calls them. Like building a website — you don't integrate with Chrome.
Do we integrate with each AI?No. We expose endpoints. AI platforms call us. We just register on their portals (like listing on Google Maps).
Does every AI use UCP?No. Google = UCP, OpenAI = ACP, Microsoft = Copilot. But 90% of the work is the same — just different field names.
Where does user context live?Google side: identity + payment (Google Wallet). Merchant side: orders + loyalty. Bridged via OAuth2 identity linking.
What if user isn't logged in?Can browse products. Can't checkout. Degrades gracefully to merchant's website.

What the Accelerator Does vs Does NOT Do
What It Does
  • Expose UCP / ACP / MCP endpoints
  • Translate field names between protocols and backend
  • Route requests to SAP OCC REST APIs
  • Track session-to-cart mapping (Redis)
  • Handle OAuth2 identity linking flow
  • Return responses in correct protocol format
What It Does NOT Do
  • Calculate pricing (SAP does this)
  • Apply discounts or promotions (SAP does this)
  • Validate inventory (SAP does this)
  • Determine shipping costs (SAP does this)
  • Store customer data (SAP/CRM does this)
  • Make any business decisions

OpenAPI / RAML definitions for UCP, ACP, MCP — maintained as Google/OpenAI update specs. Client doesn't read 200-page protocol docs.
2
Canonical Data Model
Standardized internal field names that sit between all protocols and all backends. Add new protocol = +1 mapping, not N.

Mapping Sheets
Field-by-field transformation rules. UCP ↔ Canonical ↔ SAP. ACP ↔ Canonical ↔ SFCC. The core IP.

6
Test Suite
Protocol compliance tests + AI agent simulator (Gemini/ChatGPT bots that test the full checkout flow). Validate in hours.

Leadership Q&A:
Anticipated Questions
Answers to the questions leadership will ask.
Protocol & Market Questions
QuestionAnswer
Is UCP a Google-only thing? Open standard, but Google drives it. OpenAI has ACP (similar). Both are emerging. Wipro's kit supports all — we don't bet on one.
Will one protocol win? Too early. Like VHS vs Betamax. Our approach is protocol-agnostic — the canonical data model decouples us from any single protocol.
What about Anthropic / Claude? Anthropic created MCP (Model Context Protocol) which is used as a transport INSIDE UCP. They haven't launched a commerce protocol yet. We support MCP as an endpoint.
What if the client already has Shopify? Shopify is building native UCP support. But many SAP/SFCC clients need custom integration — that's our market.
Technical Questions
QuestionAnswer
Do we need to integrate with each AI platform separately? No. We expose endpoints, they call us. Like building a website — you don't integrate with each browser. We just register on each platform's portal (form filling, not coding).
What about security? Payment NEVER touches our layer (Google Pay / Stripe handles it tokenized). We're a pass-through for product/cart data only. OAuth2 for identity linking. No PII stored.
Do we touch the client's SAP Commerce? No. We only CALL their existing SAP OCC REST APIs. Zero changes to SAP configuration, pricing rules, or business logic.
What if protocols change? That's the maintenance contract. We update mapping sheets and endpoints. ~0.5 FTE ongoing per client.
Why not just use MuleSoft? If the client has it, we use it. But buying a $100K/year license for 5 REST endpoints is overkill. SAP OCC is already REST — we don't need an enterprise integration platform.
Business Questions
QuestionAnswer
Can't SAP build this themselves? They will eventually. Their roadmap is 12-18 months out. Clients need it NOW. We bridge the gap. First-mover advantage.
What's Wipro's IP? API specs, canonical data model, mapping sheets, flow templates, config generators, test suites. The knowledge encoded in these artifacts — not tied to any platform.
ROI for the client? New sales channel. AI commerce projected $385B by 2028. Cost: ~$150K one-time + $6K/month. One premium product sale through AI covers a month of maintenance.
What's the risk if we don't do this? Competitors who ARE on UCP/ACP will capture AI-driven purchases. The client becomes invisible in the fastest-growing shopping channel.
How do we scale this as a practice? First client builds the kit. Each subsequent client is 60-70% cheaper (config-only for same backend). Every Wipro retail/CPG client is a prospect.
